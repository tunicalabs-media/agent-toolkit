## What This Skill Does
API and Data Patterns documents how to structure data fetching and API work in a Next.js App Router project. It emphasizes server-first fetching, typed responses, resilient data access, and clean boundaries between route handlers, service functions, Server Components, and Client Components.

## Features
- Layering guidance for choosing route handlers, services, and server-rendered data access.
- Rules for explicit response types, resilient fallbacks, and predictable error behavior.
- Verification checklist for data-driven pages and API integrations.

## Typical Use Cases
- Adding route handlers or service functions to a Next.js project.
- Refactoring mixed client/server data access into clearer boundaries.
- Reviewing data-fetching code for build stability and type safety.
