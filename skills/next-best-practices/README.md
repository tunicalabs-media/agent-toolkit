## What This Skill Does
Next Best Practices is a modular Next.js reference pack for writing and reviewing modern App Router code. It covers file conventions, RSC boundaries, async APIs, runtime selection, directives, route handlers, metadata, image/font optimization, scripts, hydration, Suspense, parallel routes, self-hosting, and debugging.

## Features
- Topic-specific references for common Next.js implementation and review tasks.
- Coverage of Next.js 15+ async APIs and Next.js 16 middleware/proxy naming changes.
- Practical guidance for routing, metadata, optimization, deployment, and error handling.

## Typical Use Cases
- Reviewing a Next.js route, layout, API route, or Server Component boundary.
- Migrating code to newer Next.js async APIs or file conventions.
- Debugging hydration, Suspense, metadata, image, font, or bundling issues.

## Files Included
- `SKILL.md` - Entry point that maps topics to reference files.
- `*.md` topic files - Focused references for routing, data, metadata, images, fonts, scripts, self-hosting, and related concerns.

## Notes
This skill is a local reference index. It is especially useful when the agent needs a specific Next.js rule without loading a large general-purpose guide.
